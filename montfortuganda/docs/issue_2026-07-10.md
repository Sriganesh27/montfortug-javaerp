# Enterprise Root Cause Analysis Report

## SECTION 1
### Root cause(s)

1. **Departments API 500 Error (`SQLSyntaxErrorException`):**
   The `GET /api/departments` endpoint fails because of an Entity-Database schema mismatch in the `ErpEmployee` entity. The custom JPQL query in `ErpDepartmentRepository` attempts to count employees using a subquery (`emp.active = true`). The `ErpEmployee` Java entity declares an `active` boolean column, but the actual MySQL database table `erp_employees` lacks this column (it uses `employment_status` instead). This causes MySQL to throw an "Unknown column" error, resulting in a 500 crash.

2. **Admissions API 500 Error (`BranchNotAssignedException`):**
   The `GET /api/admission/branch/applications` endpoint fails because of an unhandled custom exception. The service mandates a valid `branchId` for the requesting user via `BranchAccessService`. If the user (e.g., a Super Admin or test user) has a `null` branch ID in their session context, it throws a `BranchNotAssignedException`. Because this custom exception is not registered in the `GlobalExceptionHandler`, Spring Boot defaults to returning a 500 Internal Server Error instead of a graceful 400 Bad Request or 403 Forbidden.

3. **Frontend JavaScript Crashing (`TypeError: Cannot read properties of null`):**
   The Designations UI crashes because `DesignationView.js` is attempting to bind `addEventListener` to HTML elements (specifically `ba-searchDept` and `edit-deptId`) that were intentionally removed from the DOM during a previous UI refactoring.

## SECTION 6
### Stack trace explanation

**For `/api/departments`:**
1. Browser requests `GET /api/departments?page=0...`
2. `DepartmentController` converts URL parameters and calls `DepartmentService.searchDepartments`.
3. Service calls `ErpDepartmentRepository.searchDepartments` executing the JPQL `@Query`.
4. Hibernate translates the JPQL subquery to SQL: `SELECT count(emp.employee_id) FROM erp_employees emp WHERE emp.department_id = d.department_id AND emp.active = 1`.
5. MySQL rejects the query: `java.sql.SQLSyntaxErrorException: Unknown column 'emp.active' in 'where clause'`.
6. Spring translates the SQL error to a 500 HTTP response.

**For `/api/admission/branch/applications`:**
1. Browser requests `GET /api/admission/branch/applications`.
2. `BranchAdmissionController` extracts `Authentication` and passes it to `BranchAdmissionServiceImpl`.
3. Service calls `BranchAccessService.getValidatedBranchId(ctx)`.
4. `ctx.getBranchId()` returns `null`.
5. Method throws `BranchNotAssignedException`.
6. Exception propagates up to the DispatcherServlet. Since `GlobalExceptionHandler` lacks an `@ExceptionHandler(BranchNotAssignedException.class)`, it is treated as a fatal unhandled server exception (500).

## SECTION 7
### Minimal code changes required

1. **Database Schema Fix**: 
   Run `ALTER TABLE erp_employees ADD COLUMN active TINYINT(1) NOT NULL DEFAULT 1;` in the database to align with the `ErpEmployee` entity and satisfy the department query.
   
2. **Exception Handling Fix**:
   Add the following method to `GlobalExceptionHandler.java`:
   ```java
   @ExceptionHandler(BranchNotAssignedException.class)
   public ResponseEntity<ApiResponse<Void>> handleBranchNotAssigned(BranchNotAssignedException ex) {
       return ResponseEntity.status(HttpStatus.FORBIDDEN)
               .body(ApiResponse.error(ex.getMessage()));
   }
   ```

3. **Frontend JS Fix**:
   Remove lines 11 and 26 from `DesignationView.js` that reference the deleted department dropdowns.
